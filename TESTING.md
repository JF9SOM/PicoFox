# テスト手順書 / Testing Guide

---

## 1. RTL-SDRでの受信確認手順

### 必要機材
- RTL-SDR（RTL2832U系 USB SDRドングル）
- 145MHz帯対応アンテナ（1/4λ垂直 ≒ 51.5cm）
- PC: SDR# / GQRX / SDR++ いずれか

### 手順

**1-1. ソフトウェアセットアップ**
```
SDR# (Windows): sdrsharp.com からダウンロード
GQRX (Linux/Mac): sudo apt install gqrx-sdr
SDR++ (全OS): github.com/AlexandreRouma/SDRPlusPlus
```

**1-2. キャリア確認**
1. RTL-SDRを接続し、ソフトを起動
2. 周波数を `145.660 MHz` に設定
3. モード: `FM Narrow (NFM)`、帯域幅: `12.5 kHz`
4. PicoFoxに `CALLSIGN=JA1XXX`（任意）、`FOX_NUMBER=1` を設定して電源投入
5. 電源投入から **0秒以内** にキャリアが見えることを確認
6. 約10秒後にモールスID（MOE JA1XXX）が聞こえることを確認
7. 12秒でキャリアがOFFになることを確認

**1-3. スロットタイミング確認（単体）**
- SDR#のウォーターフォール表示で `FOX_NUMBER=1`〜`5` 各設定の送信開始時刻をストップウォッチで計測
- 期待値: Fox N の送信開始 = 電源投入から `(N-1) × 12秒`

## 2. 5局同時起動のスロット確認方法

### 事前準備
- PicoFox（またはシミュレーション用PC）を5台用意
- 各機に `FOX_NUMBER=1`〜`5` を設定（周波数はすべて同一: 145.660 MHz）

### シミュレーション方法（実機1台でテスト）
1台のPicoFoxで `FOX_NUMBER=1` から `5` に順次変更し、各スロット開始時刻を計測して確認する。

### 実機5台での確認手順

**2-1. 同時投入**
1. 5台を電源スイッチOFFの状態でUSB給電スタンバイ
2. 5台のスライドスイッチを**できるだけ同時**にON（±1秒以内が目標）
3. RTL-SDRのウォーターフォールを録画（OBS等）

**2-2. 確認ポイント**
```
秒数    期待動作
 0〜12  Fox1(MOE)のみ送信
12〜24  Fox2(MOI)のみ送信
24〜36  Fox3(MOS)のみ送信
36〜48  Fox4(MOH)のみ送信
48〜60  Fox5(MO5)のみ送信
60〜    Fox1(MOE)に戻る
```

**2-3. スロット重複の確認**
ウォーターフォール上で2局以上のキャリアが同時に見えた場合はスロット重複。  
原因: 電源投入タイミングのズレ（±1秒超）または `FOX_NUMBER` 設定ミス。

## 3. tinySAでのスプリアス測定手順

### 必要機材
- tinySA Ultra（推奨）またはtinySA Basic
- 50dBアッテネータ（直接接続時の保護用）
- SMA-SMAケーブル

### 手順

**3-1. 基本スプリアス測定**
1. PicoFoxのアンテナ端子に **50dBアッテネータ** を接続
2. アッテネータ出力をtinySAのIN端子に接続
3. tinySAの設定:
   - SPAN: `100 MHz〜500 MHz`（高調波確認）
   - RBW: `30 kHz`
4. PicoFoxを送信状態にして測定
5. 基本波（145.660 MHz）と第2高調波（291.32 MHz）、第3高調波（436.98 MHz）の電力差を確認

**3-2. 合否基準（電波法 第7条関係）**
- 基本波から **-60 dBc 以下**（スプリアス発射の強度の許容値）

**3-3. BPFありの比較測定**
HobbyPCB 144MHz BPFを挿入した状態で同様に測定し、スプリアス改善量を確認。

## 4. DRA818V到着後の初期確認手順

### Step 1: 単体動作確認（RP2040なし）

1. DRA818VのVCC=3.3V、GND接続
2. PDNをHIGHに固定
3. PCのUSB-シリアルアダプタ（3.3V）でRXD/TXDに接続
4. ターミナルソフト（Tera Term等）で `9600bps 8N1` に設定
5. 送信: `AT+DMOCONNECT\r\n`
6. 受信: `+DMOCONNECT:0` が返ればOK

**確認コマンド一覧**
```
AT+DMOCONNECT                          → ハンドシェイク確認
AT+DMOSETGROUP=1,145.6600,145.6600,0000,0,0000  → 145.660MHz / 12.5kHz設定
AT+DMOSETGROUP=0,145.6600,145.6600,0000,0,0000  → 145.660MHz / 25kHz設定
```

### Step 2: PTT動作確認

1. PTTピンをGNDに接続（LOW固定）
2. RTL-SDRで145.660 MHzにキャリアが見えるか確認
3. PTTをHIGHに戻してキャリアが消えるか確認

### Step 3: 音声変調確認

1. ファンクションジェネレータまたはスマートフォンの信号音（1kHzトーン）をMICピンに入力
2. PTT LOW（送信状態）
3. RTL-SDRのFMデモジュレータで1kHzトーンが聞こえるか確認
4. 音量を調整してFM偏移が ±2.5kHz（12.5kHz帯域幅モード）に収まるか確認

### Step 4: RP2040との統合確認

1. [`HARDWARE.md`](HARDWARE.md) の配線表に従って接続
2. `dra818_init()` を呼び出してシリアルモニタに初期化ログを確認
3. `dra818_set_group(145.660, DRA818_BW_12K5, 0)` を呼び出して周波数設定確認
4. `dra818_ptt_on()` でキャリア確認、`dra818_ptt_off()` でキャリア消滅確認
5. PWM音声出力からモールスIDが正常に変調されるか確認

## 5. 問題発生時のトラブルシューティング

| 症状 | 確認項目 |
|------|---------|
| キャリアが出ない | isConfigured フラグ確認、CALLSIGN設定確認 |
| スロットがずれる | 全局同時投入できているか、millis()オーバーフロー確認 |
| DRA818Vが応答しない | UART TX/RXの逆接確認、PDN=HIGH確認、ボーレート確認 |
| モールスIDが聞こえない | callsign.wavが生成されているか、USBマウント中でないか |
| スプリアスが多い | BPFを挿入、Si5351駆動レベルを `SI5351_DRIVE_2MA` に維持 |
| 送信電力が低い | ATTENUATION=0 確認、DRA818V VCC電流確認（700mA必要） |
